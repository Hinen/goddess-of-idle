# GODDESS OF IDLE 게임 기획서

> 승리의 여신: 니케 팬게임을 전제로 한 비공식 데스크톱 방치형 RPG 프로토타입 기획서

## 문서 정보

| 항목 | 내용 |
| --- | --- |
| 문서 상태 | 기초 프로토타입 기획 |
| 현재 버전 | 0.2.0 |
| 최종 수정일 | 2026-07-13 |
| 기준 플랫폼 | Windows PC |
| 화면 형태 | 데스크톱 화면 하단 상주형 투명 오버레이 |

이 문서는 현재 프로토타입에 필요한 핵심 전투만 다룬다. 성장, 장비, 파밍, 별도 게임 모드와 장기 콘텐츠는 기초 전투가 검증된 뒤 추가한다.

표시 규칙은 다음과 같다.

- **[확정]** 현재 프로토타입의 고정 규칙
- **[제안]** 프로토타입 제작을 위해 임시로 제안한 규칙
- **[미정]** 후속 논의가 필요한 규칙

## 1. 게임 정의

### 1.1 한 줄 소개

화면 하단에 배치된 세 명의 니케가 사용자의 평범한 키보드와 마우스 활동에 반응해 지속적으로 출현하는 랩쳐를 막는 데스크톱 방치형 RPG.

### 1.2 핵심 플레이

- 니케는 화면 하단의 고정된 전투 위치에 배치된다. **[확정]**
- 맵이나 카메라가 좌우로 전진하지 않는다. **[확정]**
- 랩쳐는 현재 스테이지의 출현 규칙에 따라 계속 나타난다. **[확정]**
- 지원형 니케는 자동으로 공격한다. **[확정]**
- 마우스 활동은 마우스 니케의 공격을 발생시킨다. **[확정]**
- 키보드 활동은 키보드 니케의 공격을 발생시킨다. **[확정]**
- 공격 대상, 스킬, 버스트, 엄폐, 재장전은 모두 자동으로 처리한다. **[확정]**
- 사용자가 전투를 위해 직접 누르는 공격, 스킬 또는 버스트 버튼은 없다. **[확정]**

### 1.3 프로토타입 핵심 흐름

```text
스테이지 시작
    ↓
랩쳐 지속 출현
    ↓
지원형 자동 공격 + 키보드·마우스 활동 공격
    ↓
자동 버스트 순환
    ↓
모든 랩쳐 처치 시 다음 스테이지
    ↓
일정 스테이지마다 세이프 포인트 저장

방어선 HP 0
    ↓
게임 오버 연출
    ↓
최근 세이프 포인트에서 재시작
```

## 2. 고정 3인 스쿼드

스쿼드는 지원형, 마우스, 키보드 슬롯으로 구성된 3인 구조다. 각 슬롯의 입력 방식, 버스트 단계, 무기군은 고정된다. **[확정]**

| 슬롯 | 공격 발생 조건 | 버스트 단계 | 사용 무기 |
| --- | --- | --- | --- |
| 지원형 니케 | 일정 주기의 자동 공격 | 버스트 I | SMG, SG |
| 마우스 니케 | 마우스 클릭 및 휠 활동 | 버스트 II | SR, RL |
| 키보드 니케 | 키보드 Key Down 활동 | 버스트 III | AR, MG |

### 2.1 지원형 니케 [확정]

- 세 명 중 유일하게 사용자 입력 없이 일반 공격한다.
- SMG 또는 SG만 사용한다.
- 버스트 I을 담당한다.
- 자동 공격의 화력과 버스트 게이지 생성량은 키보드·마우스 공격보다 낮다.
- 사용자가 입력하지 않을 때의 전투는 지원형 니케 한 명의 일반 공격에 의존한다.

### 2.2 마우스 니케 [확정]

- SR 또는 RL만 사용한다.
- 버스트 II를 담당한다.
- 마우스 클릭 또는 휠 활동이 감지되면 공격 큐에 공격을 추가한다.
- 마우스 이동은 공격으로 처리하지 않는다.
- SR은 단일 대상 강공, RL은 범위 공격이라는 차이를 갖는다.

### 2.3 키보드 니케 [확정]

- AR 또는 MG만 사용한다.
- 버스트 III를 담당한다.
- 키보드 Key Down 활동이 감지되면 공격 큐에 공격을 추가한다.
- 실제로 어떤 키가 입력됐는지는 전투 결과에 영향을 주지 않는다.
- AR과 MG의 연사속도에 맞춰 공격 큐를 순차적으로 소비한다.

## 3. 전투 입력과 자동화

### 3.1 외부 입력의 의미 [확정]

키보드와 마우스 입력은 플레이어가 게임을 향해 내리는 전투 명령이 아니다. 사용자가 다른 프로그램에서 평범하게 PC를 사용하는 활동을 감지해 담당 니케의 일반 공격을 발생시키는 장치다.

### 3.2 공격 처리 [확정]

```text
키보드 또는 마우스 활동 감지
    ↓
담당 니케의 공격 큐 증가
    ↓
AI가 랩쳐 타깃 선정
    ↓
무기 연사속도에 맞춰 공격 실행
    ↓
명중 및 대미지 처리
    ↓
버스트 게이지 충전
```

- 입력 하나마다 Spine 사격 애니메이션을 처음부터 재생하지 않는다.
- 입력이 무기 연사속도보다 빠르면 공격 큐에 잠시 저장한다.
- 공격 큐는 무한히 쌓이지 않도록 최대치와 보관 시간을 갖는다.
- 타깃 선정, 재장전, 엄폐와 스킬 사용은 AI가 처리한다.
- 전투 HUD는 상태만 표시하며 클릭 가능한 전투 버튼을 제공하지 않는다.

### 3.3 완전 자동 전투 [확정]

다음 항목에는 수동 발동이나 수동 선택이 존재하지 않는다.

- 일반 스킬
- 버스트 I, II, III
- 풀 버스트 진입
- 엄폐와 엄폐 해제
- 재장전
- 공격 대상
- 보스 파괴 부위
- 스테이지 시작 및 다음 스테이지 진입

## 4. 버스트 시스템

### 4.1 고정 순서 [확정]

```text
일반 공격 명중
    ↓
공용 버스트 게이지 충전
    ↓
지원형 니케: 버스트 I 자동 발동
    ↓
마우스 니케: 버스트 II 자동 발동
    ↓
키보드 니케: 버스트 III 자동 발동
    ↓
풀 버스트 자동 진입
```

- 버스트 게이지는 공격이 랩쳐에게 명중할 때 충전된다.
- 게이지가 가득 차고 세 버스트가 준비되면 버스트 I부터 자동 발동한다.
- 하나 이상의 버스트가 쿨타임이면 게이지를 보존하고 준비되는 즉시 자동 발동한다.
- 버스트 전용 버튼, 단축키, 프리셋과 발동 순서 설정은 없다.

### 4.2 니케별 버스트 효과 [확정]

- 버스트 스킬의 효과와 연출은 해당 니케의 고유 스킬에 따라 달라진다.
- 버스트 I, II, III에 공통된 공격, 버프 또는 방어 효과를 임의로 고정하지 않는다.
- 풀 버스트 진입 시 세 니케가 동시에 엄폐에서 나오거나 집중 사격한다는 공통 규칙을 두지 않는다.
- 풀 버스트 중 실제로 발생하는 효과와 행동은 편성된 니케들의 고유 버스트 설계를 따른다.
- 프로토타입에서는 버스트 시스템의 자동 순환과 캐릭터별 효과 연결 구조까지만 구현한다.

### 4.3 방치 중 버스트 [제안]

- 입력이 없을 때는 지원형 니케의 자동 공격만 버스트 게이지를 충전한다.
- 지원형의 게이지 생성량이 낮으므로 방치 중 버스트 발동 주기는 매우 길다.
- 방치 중에도 게이지가 가득 차면 버스트 I, II, III와 풀 버스트는 자동으로 진행한다.
- 방치 전투의 전체 평균 효율이 활성 전투의 10~20%를 넘지 않도록 자동 공격과 버스트 주기를 조정한다.

## 5. 스테이지 구조

### 5.1 고정 화면 스테이지 [확정]

- 니케와 카메라는 이동하지 않는다.
- 각 스테이지는 랩쳐의 종류, 수량, 출현 순서로 구성된다.
- 스테이지에 설정된 랩쳐가 순서대로 지속 출현한다.
- 출현 예정 랩쳐와 현재 랩쳐를 모두 처치하면 스테이지를 클리어한다.
- 클리어 후 짧은 전환 연출을 거쳐 다음 스테이지가 자동 시작된다.
- 시간 단위의 공세 정산 시스템은 사용하지 않는다.

### 5.2 세이프 포인트 [확정]

- 여러 스테이지 사이에 세이프 포인트가 존재한다.
- 세이프 포인트에 도달하면 재시작 위치가 자동 저장된다.
- 정확히 몇 스테이지마다 세이프 포인트를 둘지는 프로토타입 테스트 후 결정한다.
- 첫 세이프 포인트에 도달하기 전 게임 오버가 되면 스테이지 1에서 다시 시작한다.

### 5.3 방어선 HP와 게임 오버 [확정]

- 랩쳐의 공격이 성공하면 방어선 HP가 감소한다.
- 방어선 HP가 0이 되면 즉시 게임 오버 상태가 된다.
- 비상 정비, 자동 복구, 화력 감소 상태는 사용하지 않는다.
- 게임 오버 연출이 끝나면 최근 세이프 포인트의 첫 스테이지로 자동 복귀한다.
- 세이프 포인트 이후 현재 스테이지까지의 진행 상황은 초기화된다.
- 게임 오버 후에도 사용자가 재시작 버튼을 누를 필요는 없다.

### 5.4 게임 오버 연출 [제안]

프로토타입에서는 다음과 같은 짧은 연출만 사용한다.

1. 방어선 HP가 0이 된다.
2. 전투와 랩쳐 움직임이 정지한다.
3. 니케의 패배 또는 전투 불능 애니메이션을 재생한다.
4. `GAME OVER`와 복귀할 세이프 포인트를 표시한다.
5. 화면을 전환하고 최근 세이프 포인트에서 자동 재시작한다.

## 6. 랩쳐와 패배 조건

### 6.1 프로토타입 랩쳐 [제안]

| 유형 | 행동 |
| --- | --- |
| 일반형 | 니케를 향해 접근하고 일정 거리에서 방어선을 공격한다. |
| 장갑형 | 이동이 느리고 체력이 높으며 강한 방어선 피해를 준다. |
| 원거리형 | 먼 위치에서 공격을 준비한 뒤 방어선에 피해를 준다. |

### 6.2 기본 행동 [제안]

- 랩쳐는 스테이지 출현 데이터에 따라 생성된다.
- AI가 자동으로 공격 우선순위를 결정한다.
- 랩쳐가 공격을 완료하면 방어선 HP가 감소한다.
- 랩쳐 개별 기믹과 보스는 기본 전투가 검증된 뒤 추가한다.

## 7. Spine 및 Spring Bone

- 니케 캐릭터 애니메이션은 Spine을 사용한다. **[확정]**
- 니케에서 사용한 것과 같은 방향의 Spine Spring Bone을 사용한다. **[확정]**
- 프로토타입 기획서에서는 Spring Bone의 부위별 설정, 무기별 반동값과 세부 연출을 정의하지 않는다. **[확정]**
- 프로토타입에는 대기, 엄폐, 사격, 재장전, 버스트, 피격, 패배 상태에 필요한 최소 애니메이션만 포함한다. **[제안]**

## 8. 방치 전투

### 8.1 입력이 있는 상태 [확정]

- 지원형 니케가 자동 공격한다.
- 마우스 활동이 있으면 마우스 니케가 공격한다.
- 키보드 활동이 있으면 키보드 니케가 공격한다.
- 세 니케의 공격 명중으로 버스트 게이지가 빠르게 충전된다.

### 8.2 입력이 없는 상태 [확정]

- 지원형 니케만 자동 공격한다.
- 마우스 니케와 키보드 니케는 일반 공격하지 않는다.
- 지원형은 SMG 또는 SG를 사용하지만 주력 화력 역할이 아니므로 처치 속도가 매우 느리다.
- 버스트 게이지도 지원형의 명중으로만 천천히 충전된다.
- 방치 전투의 목표 효율은 활성 전투 대비 10~20%다.
- 방치만으로 진행할 수는 있지만 스테이지 돌파 속도와 생존력이 크게 낮아야 한다.

## 9. 프로토타입 화면

```text
┌──────────────────────── 일반 데스크톱 영역 ────────────────────────┐
│                                                                    │
│                                    랩쳐 출현 및 전투                │
│                                                                    │
├──────────────────────── 하단 전투 영역 ───────────────────────────┤
│ [지원형 B1]       [마우스 B2]       [키보드 B3]                   │
│ 스테이지 / 최근 세이프 포인트 / 방어선 HP / 버스트 상태           │
└────────────────────────────────────────────────────────────────────┘
```

프로토타입 HUD에는 다음 정보만 표시한다. **[제안]**

- 현재 스테이지
- 최근 세이프 포인트
- 방어선 HP
- 버스트 게이지와 현재 버스트 단계
- 세 니케의 재장전 및 버스트 쿨타임

## 10. 기초 프로토타입 범위

### 10.1 포함 [확정]

- 화면 하단 고정 전투 영역
- 지원형 B1, 마우스 B2, 키보드 B3의 3인 스쿼드
- 지원형 SMG/SG, 마우스 SR/RL, 키보드 AR/MG 무기 제한
- 지원형 니케의 자동 일반 공격
- 키보드 및 마우스 활동 감지
- 입력 장치별 공격 큐
- AI 자동 타깃과 재장전
- 공용 버스트 게이지
- 버스트 I → II → III → 풀 버스트 자동 순환
- 니케별로 다른 버스트 효과를 연결할 수 있는 구조
- 스테이지별 랩쳐 출현
- 방어선 HP
- 게임 오버 연출
- 최근 세이프 포인트 자동 재시작
- 최소 Spine 애니메이션
- Spine Spring Bone 사용

### 10.2 제외 [확정]

- 캐릭터 성장
- 장비 성장
- 파밍 및 드롭 아이템
- 빌드 시스템
- 오프라인 보상
- 별도 게임 모드
- 보스전
- 특수 경보
- 집중 작전
- 수동 공격
- 수동 타깃
- 수동 스킬 및 버스트
- 상세 Spring Bone 연출 기획
- 장기 콘텐츠

### 10.3 성공 조건 [제안]

1. 지원형 니케가 입력 없이 자동 공격한다.
2. 마우스 활동과 키보드 활동이 서로 다른 니케의 공격으로 즉시 연결된다.
3. 빠른 입력에도 사격 애니메이션이 끊기지 않는다.
4. 버스트 I, II, III와 풀 버스트가 사용자 개입 없이 순환한다.
5. 니케마다 서로 다른 버스트 효과를 적용할 수 있다.
6. 스테이지 클리어 후 다음 스테이지가 자동 시작된다.
7. 게임 오버 후 최근 세이프 포인트에서 자동 재시작한다.
8. 입력이 없을 때의 전투 효율이 활성 전투의 10~20% 수준이다.
9. 화면 하단 전투가 일반 PC 작업을 심각하게 방해하지 않는다.

## 11. 현재 미정 사항

1. 마우스 휠을 공격 입력으로 사용할지 여부
2. 각 니케가 화면 하단에서 차지하는 크기와 배치 순서
3. 방어선 HP의 기본값과 랩쳐 공격력
4. 세이프 포인트 사이의 스테이지 수
5. 세이프 포인트 도달 시 방어선 HP 회복 여부
6. 게임 오버 연출 시간과 패배 애니메이션 범위
7. 스테이지별 랩쳐 수와 출현 간격
8. 입력이 없는 상태로 판정하는 대기 시간
9. 프로토타입에 사용할 대표 무기 조합
